### [구현] 선형대수 글 스크롤 렉 해결 — content-visibility + 이미지 최적화 파이프라인

- **원인 분석**:
  - KaTeX 수식 201개 → span 약 7,000개 + SVG 39개의 거대 DOM (HTML 379KB). 스크롤 때마다 화면 밖 수식까지 전부 레이아웃 계산 → 전체 리레이아웃 **68ms** (프레임 예산 16ms의 4배) = 렉의 주범.
  - 본문 PNG 8장 합계 2,783KB + `width/height` 속성 없음 → 로딩 느림 + 이미지 로드 때 레이아웃 밀림(스크롤 중 덜컹).
